## 三、代理隐藏企业微信机器人 Key（重要）

### 3.1 为什么要做代理？

H5 页面是公开静态文件，任何人都可以查看源码。如果直接把企微机器人 webhook（`https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=...`）写进前端，key 会泄露，别人可以拿你的 key 随意往群里发消息。

**解决方案**：在前端和企微之间加一层代理。

- 前端只把线索 JSON 发给代理地址；
- 代理在服务端持有 `key`，把 JSON 转成企微 markdown 再推送到群；
- 客户浏览器里**没有任何 key、没有 console.log、没有 localStorage 留痕**。

### 3.2 前端已就绪

`index.html` 中已配置为代理地址占位符：

```js
const REPORT_ENDPOINT = 'https://<你的代理子域>.workers.dev/api/lead';
```

部署代理后，把这个地址替换成你的真实代理 URL 即可。

### 3.3 推荐方案 A：Cloudflare Workers（免费，无需服务器，5 分钟）

1. 注册/登录 [Cloudflare](https://dash.cloudflare.com)，进入 **Workers & Pages**；
2. 创建一个新的 Worker，例如 `zhudai-lead-proxy`；
3. 把 `proxy/cloudflare-worker.js` 的内容粘贴到 Worker 编辑器；
4. 在 Worker 的 **Settings → Variables** 中添加环境变量：
   - **Name**：`WECHAT_WEBHOOK`
   - **Value**：`https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=d20664ee-f0cf-4d77-8da5-56926fb5cd9b`
   - 勾选 **Encrypt**（加密存储）。
5. 保存并部署，得到类似 `https://zhudai-lead-proxy.<你的账户>.workers.dev` 的域名；
6. 把前端 `REPORT_ENDPOINT` 改成 `https://zhudai-lead-proxy.<你的账户>.workers.dev/api/lead`；
7. 重新 push 到 GitHub Pages。

> 如果之后换 key，只需要在 Cloudflare 后台改环境变量，不用改前端。

### 3.4 备选方案 B：Vercel Serverless Function

1. 注册/登录 [Vercel](https://vercel.com)，新建项目；
2. 上传本目录的 `proxy/vercel-api-lead.js` 到项目根目录的 `api/lead.js`；
3. 在 Vercel 项目 **Settings → Environment Variables** 中添加：
   - `WECHAT_WEBHOOK` = `https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=d20664ee-f0cf-4d77-8da5-56926fb5cd9b`
4. 部署后得到 `https://<你的项目>.vercel.app/api/lead`；
5. 把前端 `REPORT_ENDPOINT` 改成该地址；
6. 重新 push 到 GitHub Pages。

### 3.5 代理安全说明

- 代理只允许来自 `https://cathyhjw3115-sudo.github.io/zhudai-h5/` 及本地调试地址的跨域请求；
- 如果担心被恶意刷接口，可额外在代理里加 **请求频率限制** 或 **共享 token**。但共享 token 需要写进前端，依然有泄露风险，因此当前版本未加入；
- 代理会校验 `name`、`contact`、`answers` 必填，缺失字段直接返回 400。

## 六、进阶：自定义转发卡片（需公众号 JS-SDK）

当前引导层只能让用户在微信里手动转发，**转发出去的卡片是默认样式**（标题=网页标题、无缩略图）。若要带**自定义标题 / 描述 / 缩略图**，需接入公众号 JS-SDK：

1. **公众号后台**：【设置与开发 → 公众号设置 → 功能设置 → JS接口安全域名】填入你的域名；
2. **引入 SDK**：在 `index.html` 的 `<head>` 加
   ```html
   <script src="http://res.wx.qq.com/open/js/jweixin-1.6.0.js"></script>
   ```
3. **后端签名**：用 `access_token` + `jsapi_ticket` 对当前 URL 做 `sha1` 签名，向前端下发 `appId / timestamp / nonceStr / signature`；
4. **前端调用**（在 `wx.ready` 内）：
   ```js
   wx.updateAppMessageShareData({ title, desc, link, imgUrl, success(){} }); // 分享给朋友
   wx.updateTimelineShareData({ title, link, imgUrl, success(){} });          // 分享到朋友圈
   ```

> 提供公众号 **appId** 与可访问的**签名接口**后，我可以把上述调用补进 `index.html`，并把"自动弹出引导层"升级为"调起微信原生转发面板"。
